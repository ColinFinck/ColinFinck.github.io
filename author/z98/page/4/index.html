Precise Calculator Joins Open Source Support Program

by Z98

May 30, 2014

The calculator in modern versions of Windows is a very useful tool, but it is still limited in many respects especially when trying to punch in a long computation. Third parties have come up with a variety of alternatives though and it is the Project's great pleasure to present Precise Calculator as another member of the Open Source Support Program. Precise Calculator not only has feature parity with the native calculator on Windows, it offers a variety of enhancements such as unit conversions and even the ability to script calculations.

PeaZIP Joins Open Source Support Program

by Z98

May 28, 2014

The ReactOS Project is pleased another participant in the Open Source Support Program, PeaZIP. Many community members have been asking that some sort of zip utility be bundled with ReactOS for convenience and while participation in the program does not mean PeaZIP will be present by default, it will make it considerably easier for users to pre-stage PeaZIP so that it will be available to use once the installation of ReactOS is complete.

PWCT Joins Open Source Support Program

by Z98

May 28, 2014

The ReactOS Project is pleased to announce PWCT as a new participant in the Open Source Support Program. Programming Without Coding Technology (PWCT) is a set of tools that tries to make GUI programming more accessible, especially for people with little programming experience. PWCT is focused on visual programming  and comes with a very practical syntax editor. Its program paradigm ranges from imperative to event driven and uses tree hierarchies instead of a drag-and-drop style.

On installers

by Z98

May 19, 2014

The ReactOS installer is probably one of the oldest extant chunks of code in ReactOS that has not undergone significant overhauls for a very long time. Besides some tuning by Alex Ionescu to reduce resource usage and adding initial support for removable devices by Cameron Gutman, it has not seen any real major changes since Eric Kohl put it together all those years ago. For the developers, this was not too much of an issue since the installer did its job perfectly fine for the most part.
