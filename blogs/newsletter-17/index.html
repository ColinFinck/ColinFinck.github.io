Newsletter 17

by samwise52 | February 12, 2007

Busy Times

It's been a hectic 3 weeks, yes I am 1 week overdue. I've been real busy with school and personal stuff, but I haven't been the only one thats been busy. It seems in my absence that 0.3.1 is finally shaping up. In this issue I will expand on the developments that have occurred not only in the main development branch (trunk), but also in the "stable" release branch for 0.3.1.

I will also like to remind everyone of several important issues, which will be discussed a bit deeper later on. Mainly the resurection of ros-bugs mailing list and the call for testers which will help us improve and expand the usability of React as a viable Operating System. Also that the fund raising campaign is over and it was a complete success. February 08, 2007 passed and we have even more money than we setout to make. Indeed everyone at the ReactOS foundation thanks all the donators. Even so if you still desire to donate do not fret, the money will be put to good use to help promote ReactOS, aquire testing hardware and other project related costs.

Project News

  • ReactOS exposed at CUSEC last week by Alex Ionescu and as I type at SCALE, where ros devs Alex Ionescu and Art Yerkes are talking about the amazing potential of ReactOS.
  • ReactOS now compiles with GCC 4.2, RosBE 0.3.5 now only includes gcc 4.1.2, although this BE is still beta. 0.3.1 even do it compiles with 4.1.2 has not been tested enough to ensure that compiler dependant bugs are not present so gcc 3.4.5 is still recommended for it.
  • Fund raising campaing is over and it is a complete success.
  • The ros-bugs mailing list is back. It was down for several months after one of those server moves/difficulties we had, but now is back up and serving everyone with the bleeding edge issues affecting ReactOS. Making for quicker responce time to regressions and common bugs.
  • New IRC channel created #reactos-testers, specifically to discuss proper testing methodology and signup to be official tester.
  • Possible reorganization of tree in the works, if approved many usermode apps would get transfered to ros-apps tree, a now mostly unmaintained tree with many testing and development applications. When a release is made this apps would also be included.

Events on Trunk

The development branch shares a lot of issues with the 0.3.1 branch, but has severa0l important diferences. The Windows 2003 compatible thread scheduler is enabled, NT compatible LPC implentation is enabled, bootvid.dll (instead of bootvid.sys), and several other "new" features.

  • cwittich - worked on gcc 4.2 fixes, regression testing framework.
  • dcote - commited several tests for the fsrtl libs and worked on several cc and fs functions
  • fireball - fixed the build several times, and also fixed several boot time regressions.
  • gedmurphy - worked on marlett font and other enhancements.
  • greatlord - worked on cmd, rbuild, calc and other stuff all around the tree.
  • hpoussin - improved code correctness, eliminating a hack or 2, and eliminating compiler warnings.
  • hyperion - improved the msvc compiler intrinsics lib for gcc .
  • ion - worked on windows 2003 thread scheduler, native dbg api, bootvid, livecd, LPC, kernel32 and other.
  • janderwald - commited several translations from bugzilla, fixed compiler warnings, improved code correctness.
  • mbosma - updated the download! application.
  • spectrolle - fixed warnings, crashes and hard coded values, also commited some translations.
  • tkreuzer - fixed bug on user32, and comctl32, and desk.cpl.
  • tretriakov - Added support for multiple keyboard layouts.
  • weiden - Fixed compilation with GCC 4.2, -O3, and several warinings. also worked on shell32, ndis, sysdm.cpl, psdk and other.

Highlights of 0.3.1 Branch

This is just a highlight of stuff thats been commited/merged into 0.3.1, it doesn't substitute a changelog, and just peaks into the many many changes and bug fixes that have occoured since 0.3.0. Most, if not all, of these fixes are merged from trunk.

  • OpenGL(mesa32.dll) working again, but only with disabled sse and 3dnow extensions.
  • Livecd works again.
  • Set focus to defpushbutton or to first WC_EDIT child window if present, like it works on windows.
  • Set screensaver timeout to 10 minutes, default value in WinXP.
  • Support LR_DEFAULTSIZE|LR_SHARED as flags in CreateIconFromResourceEx(), LoadCursorIconImage()
  • Fix a bug in NtSetTimer which was causing us to completely disregard the due time.
  • implemented pushlocks to work on GCC 3.4.5 as well as 4.1.2+ (with no perf-hit on the latter).
  • Fix handle close bug, any killed process leaked all its handles and they were never closed.
  • Fix locking bugs in guarded mutex implementation. In race conditions some operations were not re-attempted.
  • Fixed VmWare detection.
  • Merge new icon/cursor selection code, cursors and radio button drawing code.
  • Fixed some warnings.
  • merge trunk rev : 25663 and 25664 to 0.3.1 branch - update freetype.

About Bugs

Summary of activity from January 21, 2007 to February 12, 2007. Active does not mean new, but that comments or patches where added, bug was assigned or some information changed. I would also like to point out that there are many known bugs among developers, but they don't have a bugzilla entry.

  • 111 bugs were active.
  • Oldest bug closed was 1145 - "Assertion 'Class->refs == 0' failed at ./subsys/win32k/ntuser/class.c line 69"
  • 39 bugs were closed. with 8 marked duplicate, 5 invalid, and 1 works for me.
